Use Case 3 Agent Adds Requester
Primary Actor Agent
Scope ITSD
Level User Goal
Description The agent manages the information about employees who work directly for the service desk (service desk agents) and other IT employees who assist the service desk when tickets are escalated
Precondition(s) The agent must have permission to access the step to add employees
Minimal Guarantee(s) The basic information and the business impact of requester does not create
Success Guarantee(s) The basic information and the business impact of requester create
Main Success Scenario/Flow 1. The new system record employee number, first name, last name, position, department, location, work email and work phone number.
2. The new system must also store the position of the employee's manager ("reports to"), business impact, and VIP flag.
3. The default requester business impact is "Low". The VIP flag is set to false by default.
4. There should be an option to enter notes about the requester.
5. There are three permission settings. The default is "Show only their requests".
6. The values for "permission" are Show only their requests(default), Show all their department requests, and Show all their location requests.
7. There needs to be a flag to indicate if the requester is a service request approver and another to indicate if the requester is a P.O. approver.
Extensions / Alternate Flows 2.1. We ask the requester for the role of the person they report to.
2.2. Business impact, and VIP flag are based on the judgement of the person creating the requester record.
3.1. The values for "business impact" are High medium and low.
3.2. The VIP flag is used when assigning agents to tickets and prioritizing responses.

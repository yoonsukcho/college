Use Case Use Case 4 Administrator Adds Asset
Primary Actor Administrator
Scope ITID
Level User Goal
Description Create: An administrator enters the type of asset. The system checks to make sure that each asset is of a certain type and has a unique asset number.

Update: An administrator enters a different type of an asset. The system checks to make sure the asset name has not been previously used with a unique asset number.

Delete: An administrator selects a location to delete. The system checks to make sure that there are no open tickets for that location and deletes the location.
Precondition(s) The Administrator must track tangible IT assets
Minimal Guarantee(s) There is no record for the IT asset
Success Guarantee(s) The record for IT asset is given a unique identifying number
Main Success Scenario/Flow 1. The administrator opens the asset management software
2. The administrator enters a tangible IT asset
3. The administrator selects a unique asset number
4. The administrator selects the product name of the asset
5. The administrator selects the product type of the asset
6. The administrator selects the model name of the asset
7. The administrator selects the manufacturer name of the asset
8. The administrator selects the cost of the asset
9. The administrator selects the business impact of the asset
10. The administrator selects the state of the asset
11. The administrator selects the method of procurement of the asset
12. The administrator selects the depreciation method of the asset
13. The administrator selects the location and department the asset was purchased for
14. The administrator selects a state pertaining to the asset.
15. The admistrator exits the asset management software
